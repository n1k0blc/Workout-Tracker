You are a senior full-stack engineer.

Your task is to build a production-ready personal workout tracking application based on the following specifications.

---

## CONTEXT

This is a mobile-first workout tracking application designed for real-world gym usage.

Key philosophy:
- Workout plans are flexible, not strict
- Users can deviate from schedules
- The system must still track progress accurately

---

## INPUT DOCUMENTS

You must strictly follow these:

1. Business Requirements
2. Technical Requirements
3. Architecture Overview
4. Prisma Database Schema

Do NOT invent new data models or change the schema unless explicitly instructed.

---

## TECH STACK

Frontend:
- Next.js (App Router)
- TypeScript
- Tailwind CSS
- PWA support

Backend:
- Node.js with NestJS (preferred)
- REST API

Database:
- PostgreSQL

ORM:
- Prisma

Deployment target:
- Dockerized environment running on Raspberry Pi (ARM compatible)

---

## ARCHITECTURE RULES

- Use clean architecture principles
- Separate concerns:
  - controllers
  - services
  - data access (Prisma)

- Backend must be modular:
  - auth
  - users
  - workout-cycles
  - workouts
  - exercises
  - analytics

- No business logic in controllers

---

## CORE DOMAIN RULES

### 1. Blueprint vs Workout
- WorkoutBlueprint = template
- Workout = actual execution

- Blueprint updates:
  - Only affect future workouts
  - Never modify past workouts

---

### 2. Flexible Workout Execution
Users can:
- Skip workouts
- Perform workouts on different days
- Do workouts out of order
- Start free workouts

System must:
- Suggest workouts
- Never enforce schedule

---

### 3. Workout Mode
Must support:
- Dynamic exercise changes
- Adding/removing/replacing exercises
- Adjusting sets/reps/weight during workout

---

### 4. Timer System
Implement abstraction for:
- WORKOUT timer
- SET REST timer
- EXERCISE REST timer

Timers must:
- Be reliable on mobile
- Survive UI re-renders

---

### 5. Exercise System
- Pre-seeded global exercises
- User-created exercises are private

---

### 6. Analytics
Implement:
- Volume calculation
- 1RM (Epley formula)
- PR tracking

Optimize for future performance (no heavy queries in request path if avoidable)

---
